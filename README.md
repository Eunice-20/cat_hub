# Cat_Hub

L'objetif de ce projet était de mettre en pratique nos compétences acquise en programmation javascript en développant un jeux WEB. La tâche a été d'utiliser **Phaser** pour la creationde se jeux web, en s'assurant qu'elle implémente le jeu avec succès.

**Remarque** : Phaser est un framework HTML5 gratuit que utiliser pour créer des jeux Web. Il est en JavaScript et utilise WebGL, ce qui permet d'obtenir une excellente performance en 2D et 3D.

## Description
Nopi, le chat errant, entre dans la réserve du poissonnier car il est affamé. Aidez Nopi à se nourrir tout en évitant le poissonnier et les pièges (les bombes) qu'il a installés dans sa réserve.


### C'est Quoi Cat_Hub ?

Cat_hub est un jeu de réflexion où le joueur contrôle un chat à travers des obstacles et des murs de collision en le dirigeant avec les flèches.




### Quelle est le but du jeu ? 

 L'objectif est d'obtenir le score le plus élevé en évitant de se faire toucher par les bombes et les obstacles, ou de se faire attraper par le poisonnier.


### Comment jouer 

- Appuyez sur les touches fléchées de votre ordinateur pour diriger le chat entre les obstacles.

- Prenez le maximum de poissons possible pour augmenter votre score.

- Synchronisez vos déplacements pour naviguer à travers les espaces, les bombes et le poisonnier.

- Recollectez le nombre de vies possible pour ressusciter et continuer la partie.

- La partie s'arrête lorsque vous n'avez plus de vies.


## Structure Dossiers et fichiers

- **Public:** Contient les fichiers publics accessibles par le navigateur.

- **assets:** Contient les ressources graphiques et autres fichiers statiques nécessaires pour le jeu.

- **favicon:** Contient l'icône du site.

- **game.js:** Le fichier du jeu. Il contient la configuration de Phaser, la création des scènes, et les fonctions principales du jeu.

- **main.js:** Le fichier main.js configure un serveur web simple en utilisant Express, un framework pour Node.js. Il sert des fichiers statiques à partir du répertoire public et renvoie le fichier home.html lorsqu'un utilisateur accède à la racine du site. Le serveur fonctionne sur le port 3030.
